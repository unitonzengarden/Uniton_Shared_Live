## Guardrail Behavior

| Guardrail | Behavior |
|---|---|
| SHARED immutability | Fails direct changes under `docs/LAW_CLA_LLM/SHARED/laws/` unless a commit message contains `amendment-approved` |
| Lane isolation | Fails if one task changes more than one normalized Lane folder; treats `CLA_01_VULTR/` as legacy Lane_01 |
| Required deliverables | Fails if changed task files do not include report, snapshot, and audit log deliverables |
| Commit policy | Fails commits missing `[vercel skip]` |
| Topology consistency | Warns on missing SHARED_INDEX references or registered Lane folder mismatches |
| Version consistency | Warns if root `VERSION` or SHARED `VERSION` is missing; does not force equality |
| No build drift | Fails changes under `apps/`, `src/`, `backend/`, or `services/` |

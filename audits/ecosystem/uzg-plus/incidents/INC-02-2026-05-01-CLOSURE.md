## Root cause

NTS file upload mechanism shared text with chat. Token went through chat infrastructure.

## Mitigation

1. Old PAT deleted at provider (verified `401 Unauthorized` against `api.github.com/user`).
2. New PAT deployed to 3 workspaces (length 40, prefix `ghp_8d***`).
3. LAW-NTS-CREDS-PERMANENT-V1 prevents future "ask NTS" patterns.
4. Workspace boot scripts auto-load credentials.
5. GitHub Actions per-repo secrets synced (org-level not available; account is User-type).

## Lessons learned

- **KL-A-CREDS-01:** File uploads to chat = potential leak vector. Treat any uploaded text as compromised.
- **KL-A-CREDS-02:** Permanent deployment > repeated request. NTS time + leak risk both reduced.
- **KL-A-CREDS-03:** LAW-level discipline needed for credential discipline.
- **KL-A-CREDS-04:** Auto-discovery pattern (workspace → other workspaces → Actions → escalate) reduces friction and prevents premature escalation to NTS.

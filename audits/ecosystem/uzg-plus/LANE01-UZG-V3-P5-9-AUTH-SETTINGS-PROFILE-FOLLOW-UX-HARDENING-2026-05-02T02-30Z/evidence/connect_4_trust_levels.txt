=== USE CONNECT FLOW ===

ConnectActionSheetV3 component:
  1. User taps avatar in HOME/CHAT/ENTA → opens bottom sheet
  2. 4 trust levels rendered as selectable rows
  3. User selects trust level (e.g. 'resonance')
  4. selectTrust('resonance') sets selectedTrust + step='SELECTING'
  5. User taps "Gửi yêu cầu" button
  6. connect(targetUserId) calls rpc_connect_user
  7. step='SUBMITTING' → 'SUCCESS' or 'ERROR'
  8. On success: connection request created in enta_connections (status='pending')
  9. trust_level stored in enta_connection_requests.metadata JSONB

=== ACCEPT/REJECT FLOW ===

ConnectionRequestsInboxV3:
  - GET rpc_get_connection_requests(50) → pending list
  - User accepts → rpc_accept_connection(request_id) → connection.status='accepted'
  - User rejects → rpc_reject_connection(request_id) → connection.status='rejected'

=== STATE TRANSITIONS ===

connection.status lifecycle:
  pending → accepted (rpc_accept_connection)
  pending → rejected (rpc_reject_connection)
  accepted → blocked (future Sprint 5.9.x)

=== FINDINGS ===

✓ NO email confirmation required (email_verified=true immediately)
✓ Supabase project supports BOTH password and OTP auth flows

=== IMPLICATION ===

V2 deliberately uses OTP-only flow (NOT password). Supabase password auth is enabled
at the provider level but V2 user flow ignores it.

V3 Sprint 5.1 LoginForm.signInWithPassword would technically work IF user had
password set. BUT V2 users (created via OTP magiclink) have NO encrypted_password
in auth.users → "Invalid login credentials" error.

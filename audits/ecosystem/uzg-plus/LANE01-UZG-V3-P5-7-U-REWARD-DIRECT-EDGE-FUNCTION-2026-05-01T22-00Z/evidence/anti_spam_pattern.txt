=== RATE LIMITER LOGIC ===
1. On each tap: cull tapTimestamps older than 1s
2. If tapTimestamps.length >= 5 → setRateLimitTriggered(true) + setIsOnCooldown(true) + 1.5s timeout
3. UI: AntiSpamWarning toast appears
4. After cooldown: rate limit cleared, taps resume

=== DIMINISHING RETURNS LOGIC ===
1. Increment sessionTaps on each successful tap
2. If sessionTaps >= 50 → setDiminishingReturns(true)
3. UI: DiminishingReturns strip shown ("thử Quiz hoặc Task")
4. Reward estimate: 0 instead of 0-1 random
5. Server-side enforcement (Edge Function REWARD_RULES) is authoritative

=== BOT DETECTION LOGIC ===
1. Sample last 100 tap timestamps
2. Compute intervals between consecutive taps
3. Calculate variance of intervals
4. If variance < 5ms → suspect bot pattern
5. Action: 6s cooldown (4× normal)

=== EXPECTED CONSOLE OUTPUT ===
[useEnergyCore] cooldown active — tap ignored                       (when rate limited)
[useEnergyCore] rate limit triggered (>5 taps/sec)                  (initial trigger)
[useEnergyCore] bot pattern detected — cooldown                     (variance < 5)
[useEnergyCore] diminishing returns — try Quiz/Task for higher rewards  (50+ taps)

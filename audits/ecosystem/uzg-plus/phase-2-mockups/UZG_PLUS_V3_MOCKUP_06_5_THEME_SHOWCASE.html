THEME SHOWCASE — 5 Ngũ Hành Themes
Same PLUS Hub screen rendered in 5 element themes (Kim · Thủy · Mộc · Hỏa · Thổ). User experiences ENTA dominant theme by default; can override in Settings. Theme colors apply to: nav center, FAB, accents, gradients, particles, U pill, focus rings — but NOT to app icons (identity preserved).
Principle 1 · Auto + Override Theme defaults to user's ENTA dominant element (e.g. Hỏa-Mộc → Hỏa theme). User can override anytime in Settings · Theme. ENTA recalc updates default but respects manual override.
Principle 2 · Full Immersive Theme touches all chrome: nav center gradient · FAB · top bar avatar · U pill · radial backgrounds · particles · accent links · focus rings. App icon colors stay universal (preserve identity recognition).
Principle 3 · 5 Fixed Shades Each element has 1 fixed shade per V2 precedent: Kim = Light Silver · Thủy = Deep Blue · Mộc = Green · Hỏa = Warm Red · Thổ = Earth Gold. No Light/Dark toggle — theme = identity, not preference.

Mockup #6.5 — Theme Architecture · Decisions to confirm
D1 · Implementation
Single CSS variable contract: --t-primary / --t-primary-light / --t-primary-dark / --t-tint / --t-glow / --t-bg-radial-1 / --t-bg-radial-2 / --t-canvas / --t-particle. HTML root attribute data-theme="kim|thuy|moc|hoa|tho" swaps the entire palette. Zero per-component theme logic.
D2 · What changes / what does NOT
Changes: nav center · FAB · top avatar · U pill · accents · radial bg · particles · focus rings · element-specific section tints.
NOT changes: app icons (universal identity) · Ngũ Hành element colors (data integrity) · semantic colors (success/danger).
D3 · Settings flow
Default = ENTA dominant element auto-resolved from birth data. User can override in Settings → Theme picker (5 cards above). Override marked persistent. Future ENTA recalc respects override flag — does NOT auto-revert.
